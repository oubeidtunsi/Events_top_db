Il backend è il “cervello” dell’app. Sta su un computer (quello tuo o di un compagno) e si occupa di:

- registrare nuovi utenti
- farli loggare
- dare la lista degli eventi e le loro immagini
- salvare preferiti, recensioni, notifiche, ecc.

L’app sul telefono non contiene tutti questi dati, ma li chiede al backend via internet.
Quindi il backend deve essere sempre acceso mentre usate l’app, se no queste funzionalità... non funzionano.

Per chi deve AVVIARE il backend (solo una persona)
Tu hai già tutto pronto. Gli altri non devono reinstallare il backend, basta che tu lo faccia partire.

-----------------------------------------------
1. Apri il terminale dentro la cartella backend


    Vai nella cartella backend del tuo progetto da terminale su questo ide, VSC, Visual Studio Code.
    In alto a sinistra ci sono scritti "File, Edit, Scetion, View, Go, Run, Terminal, Help.
    Cliccate Terminal e selezionate "New Terminal", vi si aprirà il terminale.
    Assicuratevi di essere al seguente percorso:
    C:\Users\ifoa\Desktop\Project_work\backend>

    Per sporstarvi dentro una cartella, il comando è cd (change directory)
    cd backend per entrare in una cartella (directory)
    C:\Users\ifoa\Desktop\Project_work> cd backend = C:\Users\ifoa\Desktop\Project_work\backend 
    cd .. serve a tornare indietro nella directory
    C:\Users\ifoa\Desktop\Project_work\backend> cd .. = C:\Users\ifoa\Desktop\Project_work>
    

2. Crea l’ambiente virtuale (solo la prima volta)

    Se non c'è il venv (Controllate a sinistra se c'è una cartella chiamata venv)
    dovete scaricarla.
    Dalla posizione in cui dovreste trovarvi dal punto 1.=
    python -m venv venv
    Dopo qualche secondo apparirà una cartella venv (non toccatela).

3. Attiva l’ambiente virtuale
    Su Windows:
    venv\Scripts\activate

    C:\Users\ifoa\Desktop\Project_work\backend> venv\Scripts\activate

    Su Mac/Linux:
    (Non mi pare li usiate, ma nel caso)
    source venv/bin/activate

    C:\Users\ifoa\Desktop\Project_work\backend> source venv/bin/activate

    Dopo il comando, vedrai (venv) all’inizio della riga. Vuol dire che l’ambiente è attivo.
    (.venv) C:\Users\ifoa\Desktop\Project_work\backend> 

4. Installa le dipendenze (una volta sola)
    pip install -r requirements.txt
    Aspetta che finisca. Potrebbe volerci un paio di minuti.

5. Configura il file .env (una volta sola)
    Dentro la cartella backend c’è un file .env. Se non c’è, crealo (puoi usare il Blocco note) e scrivici:

    text
    DB_HOST=localhost
    DB_PORT=5432
    DB_NAME=project_work
    DB_USER=postgres
    DB_PASSWORD=miapassword
    SECRET_KEY=qualsiasicosa
    FLASK_ENV=development
    Importante: la password è quella che hai scelto quando hai installato PostgreSQL. Se non la sai, chiedi aiuto a chi l’ha installato.

6. Avvia il backend
    
    python app.py
    Vedrai una scritta simile a:

    text
    * Running on http://127.0.0.1:5000
    Lascia aperto questo terminale. Il backend ora è vivo.

    🌐 Far funzionare ngrok (per far arrivare il backend al telefono)
    Poiché il telefono e il computer non sono sulla stessa rete Wi‑Fi, usiamo ngrok, che crea un “ponte” temporaneo su Internet.

--------------------------------------------------------------------------------------

1. Registrarsi a ngrok (una volta sola)

    Vai su https://dashboard.ngrok.com/signup

    Crea un account (email + password).

    Dopo il login, ti daranno un comando simile a:

    ngrok config add-authtoken 1s2d3f4g5h6...
    Copialo e incollalo in un terminale qualsiasi (anche fuori dalla cartella del backend). Premi Invio.

2. Scaricare ngrok

    Scarica l’eseguibile da https://ngrok.com/download

    Estrai il file ngrok.exe in una cartella a parte (es. C:\ngrok)
    
    Aggiungetelo al desktop o alla barra delle applicazioni così farete prima ad attivarlo

3. Avviare il tunnel

    Apri un nuovo terminale (senza chiudere quello del backend).

    Vai nella cartella dove hai messo ngrok.exe:

    cd C:\ngrok

    Scrivete:
    ngrok http 5000

    Dopo qualche secondo vedrai una riga del tipo:

    text
    Forwarding   https://abc123.ngrok-free.dev -> http://localhost:5000
    Copia l’URL https://abc123.ngrok-free.dev (quello vero, non “abc123”).

4. Inserire l’URL nell’app (sul telefono)

    Aprite l’app DropBy.

    Andate su Impostazioni (dal menu laterale, dopo avere fatto accesso, lo troverete a sinistra selezionando Settings).

    Cliccate "Cambia URL backend./Change URL backend."

    Nella casella di testo, incollate l’URL copiato da ngrok e aggiungi uno / alla fine.
    Esempio: https://abc123.ngrok-free.dev/

    Selezionate  "Salva URL".

    Ora l’app prenderà i dati dal backend.
    Se il collegamento funziona, nella schermata “Spettacoli” vedrai tutti gli eventi con le locandine.

⚠️ Importante per i riavvii futuri
Il terminale del backend e quello di ngrok devono rimanere aperti.

Ogni volta che riavviate ngrok, l’URL cambia (es. https://nuovo123.ngrok-free.dev). Dovrai solo riaprire l’app, andare su Cambia URL backend, incollare il nuovo link e salvare.

I dati (utenti, preferiti, recensioni) rimangono salvati nel database, anche se spegni il computer. Al prossimo avvio, saranno ancora lì.

-----------------------------------------------------------------------------------------------------------------------------------------------

Per gli altri membri del gruppo (che non avviano il backend)
Voi non dovete fare nulla di tecnico.
Chi ha il backend acceso vi darà un link (quello di ngrok).
Voi, sul vostro telefono con l’app già installata, andate su Impostazioni → Cambia URL backend, incollate quel link (con / finale), salvate e chiudete/riaprite l’app. Fine.

 Project Work – Backend Dropby

## Cos'è stato fatto
Il backend fornisce le API REST per l'app Android.  
È scritto in **Python/Flask** e usa un database **SQLite** (un unico file `project_work.db`).

### Funzionalità implementate:

- **Autenticazione**: registrazione, login (con token JWT), profilo utente
- **Eventi**: ricerca per città, categoria, data o testo libero; dettaglio evento (con immagini)
- **Location**: elenco, dettaglio (con immagini e servizi)
- **Recensioni**: aggiunta (da utenti loggati), visualizzazione per evento
- **Preferiti**: aggiunta, rimozione, elenco personale
- **Amicizie**: richiesta, accettazione/rifiuto, lista amici
- **Proposte eventi tra amici**: un utente può proporre un evento a un amico; l'amico riceve una notifica e può rispondere con **👍 accetto**, **👎 rifiuto** o **🤔 ci devo pensare**
- **Notifiche**: elenco notifiche, segna come letta, filtro "solo non lette"
- **Parcheggi** (funzione extra): salvataggio, storico ed eliminazione

Tutti i dati sono persistenti: anche spegnendo e riaccendendo il server, le registrazioni, amicizie, preferiti e recensioni rimangono salvati.

## Dove sono i file

backend/
├── app.py # avvia il server Flask
├── database.py # gestisce la connessione al DB
├── config.py # configurazioni (es. chiave segreta)
├── schema.sql # struttura del database (SQLite)
├── seed_data.py # script per inserire dati di esempio
├── project_work.db # il database (viene creato in automatico)
├── controllers/ # logica degli endpoint (router)
├── services/ # logica di business
├── repositories/ # accesso ai dati (query SQL)
├── utils/ # utilità (es. decorator per autenticazione)
└── static/images/ # immagini (location, locandine, planimetrie)

## Come avviare il backend (solo per l'host)
### Prerequisiti (solo per chi esegue il backend)
- Python 3.10+ installato
- *(opzionale)* ngrok per accesso remoto

### Avvio normale (rete locale)

cd backend
python -m venv venv
venv\Scripts\activate   # Windows
pip install -r requirements.txt
python app.py

### Con ngrok (accesso da remoto)
bash
ngrok http 5000
